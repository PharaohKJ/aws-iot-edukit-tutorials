SD-Card

The SD Card library is a single helper API, namely Core2ForAWS_Sdcard_Init, that wraps Espressif's virtual file drivers to initialize the SD card slot. Reading/writing to an inserted SD card can use the standard C functions fopen or fprintf. Visit Espressif's virtual file system component docs for usage. Maximum supported SD is 16GB and requires the use of a FAT/FAT32 partition table and the feature's symbol must be enabled.

To enable this feature using KConfig, use the command pio run --environment core2foraws --target menuconfig from within the root of the project directoy in your PlatformIO terminal window and go to the menu Component Config --> Core2 for AWS hardware enable to set the features you want to enable.

The SD card and the screen share the same SPI bus. In order to avoid conflicts with the screen, you must take the spi_mutex semaphore, and then call spi_poll() before accessing the SD card. Once done, give the semaphore so that the display can go back to using it.

Example

The following example writes a file named hello.txt with the contents "Hello <<YOUR SD CARD'S PRODUCT NAME>>!".

#define MOUNT_POINT "/sdcard"
sdmmc_card_t* card;
esp_err_t ret;
ret = Core2ForAWS_Sdcard_Init(MOUNT_POINT, &card);
xSemaphoreTake(spi_mutex, portMAX_DELAY);
spi_poll();
FILE* f = fopen(MOUNT_POINT"/hello.txt", "w+")
fprintf(f, "Hello %s!\n", card->cid.name);
fclose(f);
xSemaphoreGive(spi_mutex);
